# 游戏规则
## 积分规则：
1、加分：拳击+2，脚踢+3

2、扣分：被打/被踢 -1，积分最低只能被扣到0

## 扣血规则：
1、拳击-8

2、脚踢-12

3、防御减伤 75%，持续时间1.5s

注：防御过程积分照样扣 1 分 （防御保护血量，不保护积分）

## v1.0.4-202605111245
1、**架构升级：从状态同步改为输入同步**
   - 旧方案：每帧发送整个状态（x/y/state/stateTimer...），数据量大且有延迟
   - 新方案：只发送用户输入（left/right/punch/kick/jump/block），对方本地执行动作
   - 优势：带宽占用减少 80%，同步更流畅，延迟更低

2、**跳跃同步问题修复**
   - 根因：客户端（蓝方）未调用 player2:update()，导致红方跳跃时蓝方看不到
   - 修复：移除 `if game_state_mqtt.is_server` 判断，双方都必须更新 player2 物理状态

3、**远程玩家移动动画优化**
   - 根因：服务器（红方）的 player2 进入完整 update 逻辑，因 inputLeft/Right=false，walk 状态立即变回 idle
   - 修复：无论服务器还是客户端，player2（远程玩家）都跳过本地输入判断，只执行跳跃物理、动画和平滑插值

4、**防两段跳机制**
   - 发送端：正在跳跃中不发送第二次跳跃消息
   - 接收端：强制重置跳跃状态，确保同步一致性

5、**跳跃过程不被状态同步打断**
   - 修复：跳跃中时，状态同步消息不覆盖 y 坐标，避免打断物理计算

6、**移动碰撞优化**
   - 从单向限制改为双向限制，双方各退 50% 距离
   - 同时向对方移动时可以互换位置（有趣的游戏机制）
